# API Specification

This document provides a comprehensive analysis of all APIs discovered within the `Jarvis` system (`d:\AI\Jarvis`). The system predominantly operates via a single monolithic FastAPI backend located in `dashboard/server.py`. 

The architecture does not expose public GraphQL, RPC, or inbound Webhooks. Integrations (e.g., Telegram, WhatsApp) operate via polling or push mechanisms as clients rather than exposing inbound webhooks. Internal communication between the dashboard and the core engine occurs in-memory via Python function calls (`_controller.process`) rather than internal HTTP requests.

---

## 1. Global Authentication

Most APIs are protected and require one of the following methods of authentication:
1. **Session Cookie**: `jarvis_session` cookie established via the `/login` endpoint.
2. **API Token**: Provided via the `X-Dashboard-Token` header. Tested against the environment variable `JARVIS_DASHBOARD_TOKEN` (or its default).

If a request fails authentication, the server typically responds with a `401 Unauthorized` status and JSON payload `{"error": "Unauthorized"}`. HTML endpoints respond with a `303 See Other` redirect to `/login`.

## 2. Readiness & Health (Unauthenticated)

### 2.1 Health Check
- **Route:** `GET /health`
- **Method:** `GET`
- **Purpose:** Lightweight readiness probe to verify system state and uptime. Intentionally unauthenticated for load balancers or health monitoring.
- **Authentication:** None
- **Request schema:** None
- **Response schema:** `HealthResponse` (JSON)
  ```json
  {
    "ok": true,
    "state": "IDLE",
    "uptime_seconds": 120.5,
    "report": {
        "is_healthy": true,
        "checks": []
    }
  }
  ```
- **Validation rules:** None
- **Error responses:** `500 Internal Server Error` if the check crashes.

## 3. UI Endpoints (HTML)

All protected UI endpoints expect a valid session or token. If invalid, they redirect to `/login` via HTTP 303.

- `GET /login` — Serves the login page.
- `POST /login` — Authenticates user via Form Data (`username`, `password`) and sets `jarvis_session` cookie. Redirects to `/` on success.
- `GET /logout` — Clears the session cookie and redirects to `/login`.
- `GET /` — Core dashboard interface.
- `GET /memory` — Memory search view. Accepts query param `q`.
- `GET /goals` — Displays active and past goals.
- `GET /search` — Advanced local file searching interface.
- `GET /converter` — Universal file conversion interface.
- `GET /health-ui` — Detailed UI for health reports.
- `GET /clicker` — GUI controller for the Auto-Clicker module.
- `GET /ai-os` — Blueprint and plugins catalog page.
- `GET /gui-audit/{filename}` — Serves screenshots generated by the GUI audit system. 
  - **Validation:** Path validation prevents directory traversal out of the `outputs/gui_audit` directory. Yields `403 Forbidden` on access violation.

## 4. Protected JSON APIs

### 4.1 Command Execution
- **Route:** `POST /command`
- **Method:** `POST`
- **Purpose:** Submit natural language instructions or commands to the Jarvis core engine.
- **Authentication:** Required
- **Request schema:** `CommandRequest` (JSON)
  ```json
  {
    "text": "Check my unread emails"
  }
  ```
- **Response schema:** `CommandResponse` (JSON)
  ```json
  {
    "response": "Found 3 unread emails...",
    "error": null
  }
  ```
- **Validation rules:**
  - `text`: Must be a string. Whitespace stripped. Min length: 1, Max length: 4096.
- **Error responses:**
  - `401 Unauthorized`
  - `422 Unprocessable Entity` (Pydantic validation failure)
  - `200 OK` with static error string mapped to `response` if processing crashes.

### 4.2 Auto-Clicker Management
The system provides tools to automate on-screen clicking tasks based on visual targets.

#### Get State
- **Route:** `GET /api/clicker/state`
- **Method:** `GET`
- **Purpose:** Retrieve the clicker's current operational status and recent logs.
- **Request schema:** None
- **Response schema:** `ClickerStateResponse` (JSON)
  ```json
  {
    "running": false,
    "target": "Submit Button",
    "interval": 5.0,
    "continuous": false,
    "min_confidence": 0.5,
    "attempts": 5,
    "successes": 4,
    "failures": 1,
    "uptime": 25.5,
    "logs": [
      {
        "timestamp": "14:20:05",
        "message": "Click FAILED",
        "level": "error"
      }
    ]
  }
  ```

#### Start Clicker
- **Route:** `POST /api/clicker/start`
- **Method:** `POST`
- **Purpose:** Initiate the auto-clicker background task.
- **Request schema:** `ClickerStartRequest` (JSON)
  ```json
  {
    "target": "SearchIcon",
    "interval": 5.0,
    "continuous": false,
    "min_confidence": 0.2
  }
  ```
- **Validation rules:**
  - `target`: String, length 1-100.
  - `interval`: Float, range 0.5 - 3600.0.
  - `min_confidence`: Float, range 0.0 - 1.0.
- **Response schema:** `GenericResponse` `{"ok": true, "error": null}`

#### Stop Clicker
- **Route:** `POST /api/clicker/stop`
- **Method:** `POST`
- **Purpose:** Halts the running auto-clicker task.
- **Request schema:** None
- **Response schema:** `GenericResponse` `{"ok": true, "error": null}`

#### Clear Logs
- **Route:** `POST /api/clicker/clear-logs`
- **Method:** `POST`
- **Purpose:** Flushes the in-memory log queue for the auto-clicker.
- **Request schema:** None
- **Response schema:** `GenericResponse` `{"ok": true, "error": null}`

#### Get Screenshots
- **Route:** `GET /api/clicker/screenshots`
- **Method:** `GET`
- **Purpose:** Fetches metadata for recent screenshots captured during GUI operations.
- **Request schema:** None
- **Response schema:** JSON Array
  ```json
  [
    {
      "name": "screenshot_01.png",
      "url": "/gui-audit/screenshot_01.png",
      "time": "2026-06-11 15:45:00"
    }
  ]
  ```

### 4.3 Goal Management
APIs to programmatically assign objectives to the agentic core.

#### Add Goal
- **Route:** `POST /goals/add`
- **Method:** `POST`
- **Purpose:** Append a new goal to the system's objective queue.
- **Request schema:** `GoalAddRequest` (JSON)
  ```json
  {
    "description": "Analyze network logs",
    "priority": 5
  }
  ```
- **Validation rules:**
  - `description`: String, length 1-4096.
  - `priority`: Integer, range 1-10.
- **Response schema:** `GenericResponse` `{"ok": true, "error": null}`

#### Complete Goal
- **Route:** `POST /goals/complete/{goal_id}`
- **Method:** `POST`
- **Purpose:** Mark a tracked goal as successfully completed.
- **Request schema:** Path parameter `goal_id` (string)
- **Response schema:** `GenericResponse` `{"ok": true, "error": null}`

### 4.4 Advanced Utilities

#### Fast Search Tool
- **Route:** `POST /api/search`
- **Method:** `POST`
- **Purpose:** Perform a high-speed multi-threaded text search across directories.
- **Request schema:** `SearchRequest` (JSON)
  ```json
  {
    "path": "all",
    "query": "*.py",
    "content": "def main",
    "threads": 8,
    "case_sensitive": false,
    "no_skip": false
  }
  ```
- **Validation rules:** If `path` is not `"all"`, it must strictly pass internal `_assert_safe_path` checks ensuring it does not traverse out of configured workspace boundaries.
- **Response schema:** JSON payload containing search results.
- **Error responses:** `403 Forbidden` if safety checks fail.

#### Universal Converter
- **Route:** `POST /api/convert`
- **Method:** `POST`
- **Purpose:** Ingest a file of arbitrary format and convert it to a specified format.
- **Request schema:** `multipart/form-data`
  - `file`: Raw file payload (`UploadFile`)
  - `target_format`: Target extension string (e.g., "pdf").
- **Validation rules:** `target_format` must match regex `^[a-zA-Z0-9]+$`.
- **Response schema:** Binary file payload (`application/octet-stream`) with `Content-Disposition` specifying the new filename.
- **Error responses:** `400 Bad Request` if invalid target format.

#### Safe File Viewer
- **Route:** `GET /api/view-file`
- **Method:** `GET`
- **Purpose:** Returns the binary stream of a requested file from the filesystem.
- **Request schema:** Query parameter `path` (string)
- **Validation rules:** Strict `_assert_safe_path` check. Constrained to `ALLOWED_DIRECTORIES`.
- **Response schema:** `application/octet-stream` containing the file content.
- **Error responses:** `403 Forbidden` on invalid path or directory traversal, `404 Not Found` if non-existent.

#### AI OS System Data
- **Route:** `GET /api/ai-os`
- **Method:** `GET`
- **Purpose:** Deliver a structural blueprint of the AI OS and loaded plugin catalogs.
- **Request schema:** None
- **Response schema:** JSON Dictionary
  ```json
  {
    "blueprint": {},
    "plugins": { "count": 1, "plugins": [], "errors": {} },
    "workflows": { "count": 0, "templates": [], "errors": {} },
    "errors": []
  }
  ```

## 5. WebSockets

- **Route:** `WS /ws`
- **Method:** `WebSocket`
- **Purpose:** Establish a persistent connection for streaming real-time status updates from the Jarvis backend to the web client.
- **Authentication:** Token provided via `?token=` query parameter, or through the `jarvis_session` cookie present during connection upgrade. Disconnects with Policy Violation (Close Code 1008) if unauthenticated.
- **Behavior:** Once connected, the server pushes a JSON payload every 2 seconds. The connection remains open infinitely until closed by the client or server.
- **Message Schema:** (Server -> Client)
  ```json
  {
    "state": "THINKING",
    "last_response": "Searching logs...",
    "last_input": "What went wrong?",
    "session_id": "ab12c3d4",
    "memory_count": 150,
    "active_goals": 2,
    "ollama_online": true,
    "uptime_seconds": 3600.2,
    "model": "llama3"
  }
  ```
