```
your-repo/                          ← root of your GitHub repository
│
├── to_commit_pool/                 ← FILES FOUND BY finder.py GO HERE
│   ├── _MANIFEST.txt               ← auto-generated by finder.py
│   ├── Users/
│   │   └── YourName/

│   │       │       └── main.py     ← example file

│   │           └── script.js
│   └── ... (mirrors your local path structure)
│
├── .github/
│   └── workflows/
│       └── daily_auto_commit.yml  ← the workflow file
│
├── finder.py                       ← the scanner script (optional, can keep local)
└── README.md                       ← optional
```

> **Key rule:** The `to_commit_pool/` folder must exist at the **root** of your repository.

## 🔐 Step-by-Step: Set Up MY_PAT in GitHub Secrets

### Step 1 — Create a Personal Access Token (PAT)

1. Go to **https://github.com/settings/tokens**
2. Click **"Generate new token" → "Generate new token (classic)"**
3. Give it a name like `MY_PAT_AUTO_COMMIT`
4. Set **Expiration** to your preference (90 days, 1 year, or No expiration)
5. Under **Scopes**, check:
   - ✅ `repo` (full control of private repositories)
   - ✅ `workflow` (update GitHub Actions workflows)
6. Click **"Generate token"**
7. **COPY THE TOKEN NOW** — you won't see it again!

---

### Step 2 — Add PAT to GitHub Secrets

1. Go to your repository on GitHub:  
   `https://github.com/Naseer-fez/<your-repo-name>`
2. Click **Settings** (top tab)
3. In the left sidebar → **Secrets and variables → Actions**
4. Click **"New repository secret"**
5. Fill in:
   - **Name:** `MY_PAT`
   - **Secret:** *(paste the token you copied)*
6. Click **"Add secret"**

✅ Done! The workflow will now use this PAT to push code.

## 🗂 How to Set Up the Repository

### Option A — Create a brand new repo

```bash
# On your Windows machine (Git Bash or PowerShell):
mkdir auto-commit-pool
cd auto-commit-pool
git init
mkdir to_commit_pool
echo "# Auto Commit Pool" > README.md
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/Naseer-fez/auto-commit-pool.git
git push -u origin main
```

### Option B — Use an existing repo

```bash
cd path/to/your/existing/repo
mkdir to_commit_pool
# Copy the workflow file:
mkdir -p .github/workflows
# Paste daily_auto_commit.yml into .github/workflows/
git add .github/workflows/daily_auto_commit.yml to_commit_pool/
git commit -m "Add auto-commit workflow and pool"
git push
```

## ⚡ Running finder.py (Windows)

### Basic run (scans Desktop, Documents, Downloads):
```cmd
python finder.py
```

### With extra directories:
```cmd
python finder.py "C:\Projects" "D:\Work\Code" "E:\Archive"
```

### What it does:
1. Scans all 3 default folders + any extra paths you pass
2. Finds every git repo and checks for dirty files
3. Finds loose code files outside any repo
4. Copies everything to `C:\Users\YourName\Desktop\to_commit_pool\`
5. Shows a colour-coded summary

### After running finder.py — push the pool to GitHub:
```bash
cd path/to/your/repo
# Copy to_commit_pool from Desktop into your repo:
xcopy /E /I /Y "C:\Users\%USERNAME%\Desktop\to_commit_pool" "to_commit_pool"
git add to_commit_pool/
git commit -m "chore: update commit pool"
git push
```

## 🧪 How to Manually Trigger the Workflow (for Testing)

1. Go to your repository on GitHub
2. Click the **"Actions"** tab
3. In the left sidebar, click **"🚀 Daily Auto-Commit from Pool"**
4. Click **"Run workflow"** (top right dropdown)
5. Select branch: `main`
6. Click the green **"Run workflow"** button
7. Watch it run live by clicking on the job

## ✅ How to Verify the Workflow is Running Correctly

### Check 1 — Actions tab
Go to `https://github.com/Naseer-fez/<repo>/actions`  
You should see runs listed with ✅ green checkmarks.

### Check 2 — Commit history
Go to the **"Commits"** tab of your repo.  
You should see commits like:
```
Add Python module: main
chore: remove main.py from pool after commit
Add JavaScript component: App
chore: update pool manifest (12 files remaining)
```

### Check 3 — Pool shrinking
Watch `to_commit_pool/` folder getting smaller each day.  
When it hits 0 files, the workflow logs:
```
✅ Pool empty — streak complete!
```

### Check 4 — Pool empty gracefully
If the pool is already empty and the workflow fires, it will:
- Print "Pool empty — streak done!"
- Exit with **success** (green ✅), never fail

## ❓ Troubleshooting

| Problem | Fix |
|---|---|
| `GITHUB_PAT not found` | Set env var on Windows (see Step 3) |
| Workflow fails with 403 | Re-check `MY_PAT` secret has `repo` + `workflow` scope |
| Pool not shrinking | Manually trigger and read the logs in Actions tab |
| finder.py crashes on a dir | That dir is permission-restricted — normal, it skips |
| git not found | Install Git for Windows: https://git-scm.com/download/win |
| ANSI colours broken | Run in Windows Terminal instead of old cmd.exe |
